Seed App

Full-Stack Starter Kit

Angular 21 · ASP.NET Core 10 · .NET MAUI · Docker

A production-ready seed application to jumpstart your next full-stack project. Clone it, rename it, and build on top of a solid architecture - without starting from scratch.

What's included

Frontend

Angular 21 SPA with Server-Side Rendering, Angular Material, and three shared libraries: shared-ui, shared-core, shared-auth.

Backend

ASP.NET Core 10 Web API following Clean Architecture. MediatR for CQRS, FluentValidation, Mapster, EF Core 10 with PostgreSQL, JWT auth, and Serilog.

Mobile

.NET MAUI application sharing business logic with the backend through the same .NET solution.

Infrastructure

Docker Compose for local development: PostgreSQL, API, and web frontend - all running with a single command.

How to use it

  1. Clone the repository

    Fork or clone this repo as the starting point for your project.

    git clone <repo-url> my-app
cd my-app
  2. Rename the solution

    Replace all occurrences of Seed with your project name across .slnx, .csproj, namespaces, and Angular project names in angular.json.

  3. Configure environment

    Copy the environment template and adjust values if needed. The defaults work out of the box for local development.

    cd docker
cp .env.example .env
  4. Start local infrastructure

    Bring up PostgreSQL and the full stack with Docker Compose.

    docker compose up --build
  5. Start building

    Add your domain entities in Seed.Domain, create MediatR commands/queries in Seed.Application, expose them via controllers in Seed.Api, and build your Angular pages.

Project structure

seed-app/
|-- backend/
|   `-- src/
|       |-- Seed.Api            <- ASP.NET Core controllers & middleware
|       |-- Seed.Application    <- MediatR handlers, validators, DTOs
|       |-- Seed.Domain         <- Entities & business logic
|       |-- Seed.Infrastructure <- EF Core, Identity, Serilog
|       `-- Seed.Shared         <- Cross-cutting utilities
|-- frontend/
|   |-- web/
|   |   `-- projects/
|   |       |-- app
|   |       |-- shared-ui
|   |       |-- shared-core
|   |       `-- shared-auth
|   `-- mobile/
`-- docker/
